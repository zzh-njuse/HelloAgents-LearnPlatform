# CS 面试备考方法论

## 答题框架

面对任何 CS 概念题，使用五步法构建回答：

```
概念定义 → 工作原理 → 核心数据/结构 → 应用场景 → 与其他概念的区别
```

### 示例：TCP 三次握手
1. **概念定义**: "TCP 三次握手是建立可靠连接的过程，通过三次报文交换确保双方收发能力正常"
2. **工作原理**: SYN → SYN-ACK → ACK，每次携带序列号
3. **核心数据**: SYN 标志位、seq 序列号、ack 确认号
4. **应用场景**: HTTP/HTTPS 请求前、WebSocket 连接建立、数据库连接
5. **对比区别**: 与四次挥手对比（建立 vs 释放）、与 UDP 对比（有连接 vs 无连接）

## STAR 法则在技术面试中的应用

| 步骤 | 说明 | 示例 |
|------|------|------|
| S - Situation | 项目背景/场景 | "在电商高并发场景下，缓存穿透导致数据库压力过大" |
| T - Task | 要解决什么问题 | "需要设计一个防止缓存穿透的方案" |
| A - Action | 你做了什么 | "我使用了布隆过滤器 + 空值缓存 + 互斥锁三重防护" |
| R - Result | 结果与量化 | "缓存命中率从 85% 提升到 99%，DB QPS 下降 60%" |

## 高频考点对比技巧

面试官经常要求"对比 A 和 B"，使用结构化对比：

| 维度 | TCP | UDP |
|------|-----|-----|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠（确认+重传） | 不可靠 |
| 顺序 | 有序 | 无序 |
| 开销 | 头部 20 字节 | 头部 8 字节 |
| 场景 | HTTP、FTP、SSH | 视频直播、DNS、游戏 |

## 面试常见陷阱

1. **概念混淆**: 进程 vs 线程、堆 vs 栈、TCP vs HTTP（不同层级）
2. **只知其一**: 说 HTTP 无状态但不知道 Cookie/Session/Token 如何补充
3. **背答案不会推导**: 被追问"为什么三次不是两次"时卡住
4. **忽略边界**: 说 HashMap O(1) 不提哈希冲突退化为 O(n)
