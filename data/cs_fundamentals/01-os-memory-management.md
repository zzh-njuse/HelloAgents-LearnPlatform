# 操作系统内存管理

## 1. 虚拟内存

虚拟内存是操作系统提供的一种内存管理技术，它为每个进程提供一个独立的、连续的虚拟地址空间，使得进程认为自己独占了整个内存。

### 核心概念
- **虚拟地址**：进程使用的逻辑地址
- **物理地址**：实际内存的硬件地址
- **页表**：维护虚拟地址到物理地址的映射关系
- **MMU**（Memory Management Unit）：硬件单元，负责地址转换

### 工作原理
1. CPU 发出虚拟地址访问内存
2. MMU 查找页表，将虚拟地址转换为物理地址
3. 如果页不在物理内存中（缺页），触发缺页中断
4. 操作系统从磁盘将缺失的页面加载到物理内存
5. 更新页表，重新执行访存指令

### 为什么需要虚拟内存
- **隔离性**：每个进程有独立的地址空间，一个进程无法访问其他进程的内存
- **扩展性**：程序可以使用比物理内存更大的地址空间
- **简化编程**：程序员不需要关心物理内存布局
- **共享**：多个进程可以共享同一物理页面（如共享库）

## 2. 分页与分段

### 分页（Paging）
将虚拟地址空间划分为固定大小的页（通常 4KB），物理内存划分为相同大小的页框（Page Frame）。

**地址转换公式**：
- 虚拟地址 = 页号 + 页内偏移
- 物理地址 = 页框号 × 页面大小 + 页内偏移

**多级页表**：
- 32 位系统常用二级页表：页目录 → 页表 → 页
- 64 位系统常用四级页表：PGD → PUD → PMD → PTE → 页
- 多级页表可以节省内存，因为不需要为未使用的地址空间分配页表

### 分段（Segmentation）
将程序划分为逻辑段（代码段、数据段、堆栈段），每段有独立的基址和界限。

### 段页式
结合分段和分页的优点：先分段，再在段内分页。Linux 主要使用分页机制。

## 3. 页面置换算法

当物理内存满且需要加载新页时，需要选择一个页面换出到磁盘。

### FIFO（先进先出）
- 策略：选择最早进入内存的页面换出
- 优点：实现简单
- 缺点：可能出现 Belady 异常（增加页框数反而增加缺页率）
- 实际应用：很少直接使用

### LRU（最近最久未使用）
- 策略：选择最长时间没有被访问的页面换出
- 优点：接近最优算法，效果较好
- 缺点：需要维护访问顺序，硬件支持成本高
- 近似实现：Clock 算法（使用访问位）

### Clock 算法（NRU）
- 维护一个循环链表 + 访问位
- 指针循环扫描：访问位=1 → 清零并跳过；访问位=0 → 换出
- Linux 使用改进版 Clock 算法

### LFU（最不经常使用）
- 策略：选择访问次数最少的页面换出
- 缺点：新加载的页面可能因访问次数少而被过早换出

### 面试高频考题
Q: 什么是颠簸（Thrashing）？如何解决？
A: 颠簸是系统花在页面置换上的时间多于执行进程的时间。原因：分配给进程的物理页框不足。解决：增加物理内存、减少并发进程数、改用局部置换策略。
