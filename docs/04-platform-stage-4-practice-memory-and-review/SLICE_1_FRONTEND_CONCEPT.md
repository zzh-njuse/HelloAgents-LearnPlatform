# Stage 4 Slice 1 前端概念

状态：已于 2026-07-16 通过人工 Gate

## 页面关系

```text
┌──────────────┬─────────────────────────────────┬──────────────────┐
│ 课节目录      │ 中间主内容                       │ 右侧辅助栏        │
│              │                                 │                  │
│ 当前课程      │ [正文] [练习]                   │ [Tutor] [练习记录]│
│ Section       │                                 │                  │
│  Lesson       │ 正文或完整练习界面              │ Tutor 对话或      │
│  Lesson       │ 题干、选项/简答、反馈与引用      │ Set/Attempt 历史  │
└──────────────┴─────────────────────────────────┴──────────────────┘
```

- 完整练习使用中间主内容区，不放进右侧窄栏。
- 中间“正文 / 练习”可随时切换；右侧“Tutor / 练习记录”可独立切换。
- 左侧当前课节是 Reader、lesson-scope Tutor、Practice 与练习记录的共享上下文；切换课节时四处必须同步，course-scope Tutor 不受课节筛选限制。
- 打开 Tutor 不得丢失当前 Practice Set、题号、未提交答案、反馈展开状态或滚动进度。
- 回到正文时 Tutor 继续绑定当前课节上下文。
- 练习记录用于选择 Practice Set、查看完成进度和历史 Attempt，不复制完整题目正文。

## 练习主界面

- 顶部显示课节、Practice Set 身份、难度、语言和题目进度。
- 中间一次聚焦一道题；长题干、长选项和简答输入不能挤压右栏或遮挡后续内容。
- 单选提交后显示用户选择、正确答案、所选项为什么错误、正确项为什么成立及资料位置。
- 简答提交后显示 AI 反馈标识、verdict、分数、rubric 分项、具体错误/遗漏、正确思路、参考讲解及资料位置。
- 底部提供上一题、下一题、重新作答和必要的删除/历史入口；命令使用清晰图标与文字。

## 状态矩阵

| 状态 | 中间主内容 | 右侧练习记录 |
|---|---|---|
| 无 Practice Set | 空状态、生成参数和“生成练习” | 空状态 |
| generating | 就地显示带任务身份的进度、取消入口 | 显示 running Set/Job |
| ready/unanswered | 题目与答题控件，隐藏评分材料 | Set 进度 0/N |
| grading | 锁定当前已提交 Attempt，显示可见任务状态 | Attempt 显示 grading |
| succeeded | 显示可信 Feedback 和资料位置 | 更新完成数量和历史 |
| failed/queue_failed | 稳定错误、重试或重新提交入口 | 显示失败身份，不显示原始错误 |
| source_degraded | 历史题目/Attempt/Feedback 只读，引用不可用；禁止新生成、作答、重做或评分 | Set 标注来源已变化且只读 |
| deleting | 立即隐藏或禁用交互，等待清理 | 不再作为可选 Set |

## 响应式边界

- Chrome 桌面为本 Slice 人工基线。
- 较窄视口优先保留中间练习；课节目录和右侧辅助栏改为可打开的独立区域，而不是压缩题目宽度。
- 不在本 Slice 承诺完整 Edge/移动端矩阵，但不得出现文字重叠、按钮溢出或无法滚动。

## Frontend Gate

GLM 必须沿用此信息架构和现有视觉系统，不重新设计导航。若实际代码结构无法同时满足“中间练习状态保留”和“右侧 Tutor 切换”，应停止对应实现并报告 Codex，不得自行选择不同布局。
