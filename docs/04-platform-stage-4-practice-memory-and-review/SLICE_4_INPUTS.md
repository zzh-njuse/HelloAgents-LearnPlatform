# Stage 4 Slice 4 输入：受控 MCP 外部工具闭环

状态：首版双 capability 已实现候选；2026-07-21 修订 Spec/ADR 已接受，进入增量任务包 002

日期：2026-07-19

## 目标

建立两个互补但受同一准入模型约束的 MCP 闭环：自托管代码执行验证本地工具服务、隔离和数据主权；远程 Wolfram 科学工具验证外部专业服务、逐 Turn 授权和 Agent 受限自主选择。两者都必须可授权、可追踪、可取消、可删除和可降级。

Slice 4 不是为了证明“平台接入了 MCP”，而是验证外部工具是否能在不破坏课程、练习、Memory 和 Tutor 权威事实的前提下提供独立学习价值。

2026-07-21 补充目标：MCP 不能停留在独立实验室或单一 Tutor 科学开关。正式闭环还包括本地公式渲染、Lesson 科学验证、Practice 编程判题/科学验证，以及 Tutor 经 Turn 授权后的自主代码与科学工具选择。

## 已有权威输入

- RAG：固定 Course/Lesson Version 下的课程事实与引用。
- Practice：题目快照、Attempt、Answer、Feedback 和评分依据。
- Slice 2：mastery、Weakness、Review Item、active Memory 与 Lesson Completion。
- Slice 3：版本化教学 Skill、Tutor Session/Turn、结构化回答、预算、trace、取消、重试和最终权威重检。
- Postgres 是产品事实来源；Qdrant 可重建；Redis 非权威；任何 MCP 返回值都不能反向成为未经确认的学习事实。

## 首轮候选与结论

### 代码执行沙箱

潜在价值：运行学习者代码、展示 stdout/stderr、测试结果和受限资源使用，让编程练习从静态讲解进入可验证执行。

主要风险：任意代码执行、文件和网络访问、资源耗尽、依赖供应链、平台差异、上传资料泄漏及复杂部署。若选择该方向，默认必须无网络、临时文件系统、严格 CPU/内存/时间限制，并明确支持的语言和包。

结论：选择。首版固定 Python、Java、C++，由用户显式触发，采用产品 MCP adapter 和成熟自托管执行后端。

### 数学、物理和通用化学科学工具

潜在价值：验证算式、符号化简、方程求解、物理量与通用化学计算，为科学学习提供可重复的专业计算依据。

主要风险：工具结果被误当作证明、表达式解析与资源耗尽、数值/符号语义差异、支持范围不清。必须区分“计算结果”“推导解释”和“课程事实”。

结论：选择 Wolfram 远程 MCP。管理员审核配置，用户按 Tutor Turn 授权，Agent 只在固定白名单和调用预算内自主选择。

### 日历与复习计划同步

潜在价值：把 Review Item 转为用户真实日程中的复习安排，并同步完成、延期或取消状态。

主要风险：第三方账号授权、个人日程隐私、双向同步冲突、撤销授权、外部删除和重复事件。内部 Review Queue 仍须保持权威，外部日历不可反向改写 mastery 或 Memory。

结论：暂缓。它会引入个人 OAuth、双向同步和远程删除，超出 Stage 4 收尾边界。

## Spec 前必须回答

1. 两项固定 capability 分别解决什么学习任务，为什么需要同时验证 self-host 与 remote MCP？
2. 为什么必须通过 MCP 或外部工具完成，内部 adapter、现有 provider 或纯前端能力为什么不足？
3. MCP server 是默认 self-host、可选部署还是第三方服务？它的版本和能力如何发现、固定与升级？
4. 谁在什么时点授权？授权是 Workspace 级、Course 级、Session 级还是单次调用级？如何撤销？
5. 哪些最小输入可以外发？是否包含用户答案、课程片段、Memory、文件或个人日程？默认禁止发送哪些内容？
6. 工具结果如何标注来源、server/version、调用时间、输入摘要、可信边界及是否经过 Tutor 解释？
7. 超时、不可用、部分成功、重复 delivery、取消、重试、晚到结果和预算耗尽如何收敛？
8. 哪些数据进入 Postgres 权威记录，哪些只保留安全摘要，哪些必须随对象删除？
9. 如何防止模型自由选择未授权工具、扩大参数范围、连续调用或把工具结果写成未经验证的学习事实？
10. 用什么离线 eval、集成测试和人工安全 smoke 证明独立价值与边界？

- 必要 ADR：server 部署模式、协议/版本固定、授权、外发、trace、队列与数据权威。
- 人工接受后，才可生成 GLM 实现任务包。

## 明确不做

- 不把双 capability 扩展成任意 MCP server、动态 Tool 或第三个未经 Gate 的场景。
- 不把 Postgres、Qdrant、Redis 或内部文件存储简单包装成 MCP。
- 不开放任意网页搜索、任意命令、任意网络访问或模型自由发现并调用未知工具。
- 不让 MCP 输出直接创建 mastery、Weakness、Memory 或完成状态。
- 不在工具选择和 Spec/ADR Gate 前修改 schema、Compose、队列或 Web 产品路径。
- 不引入自主多 Agent、认证、多租户、计费或通用插件市场。
