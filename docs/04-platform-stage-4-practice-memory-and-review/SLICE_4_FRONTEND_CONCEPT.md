# Stage 4 Slice 4 前端概念：公式、编程练习与受控工具

状态：修订版已接受（2026-07-21 人工 Gate）

> 2026-07-21 修订：原概念只覆盖独立代码实验室和 Tutor 科学开关，现扩展为统一公式渲染、Practice 编程题和 Tutor 双工具授权。旧实验室入口继续保留，但不再是编程题或 Tutor 运行代码的前置步骤。

## 0. 统一公式显示

- 课程草稿、正式 Reader、Practice 题目/反馈和 Tutor 回答统一渲染行内/块级数学公式及通用化学式/反应式。
- 使用本地固定版本 KaTeX + `mhchem`；Wolfram 不承担排版。
- 单个公式失败时局部显示原表达式与错误提示，正文其他部分继续可读。
- 块公式可横向滚动，专注页和窄视口不得因长公式撑破布局。

## 1. 代码实验室

- Reader 中间区域在“正文 / 练习”旁新增“实验室”标签。
- Workspace 未启用或 execution MCP 不可用时显示明确状态，不降级为浏览器本地执行。
- 左侧继续使用课节目录；运行跟随当前 Course/Lesson 归类，但执行服务不读取课程正文。

中间区域包含：

1. 紧凑的 Python / Java / C++ segmented control；
2. CodeMirror 6 代码编辑器：语法高亮、行号、括号匹配、Tab 缩进和稳定最小高度；
3. 固定 runtime、时间和输出上限；
4. “运行”按钮和就近任务状态；
5. “结果 / stdout / stderr / 编译”标签和退出状态、耗时、截断标志；
6. 最近运行选择与删除；
7. “用于下一次 Tutor 提问”单次复选框，默认关闭，作为用户已有运行结果的可选上下文；
8. 右上角“专注编码”图标，进入接近整页的编辑器/输出工作台。

视觉上采用安静的开发工具布局：单一工作台边界、清晰工具栏、深浅对比明确的编辑区、紧凑状态和输出 tabs；不堆叠多层卡片，不用大段裸 textarea、巨大按钮或让运行历史挤压当前代码。

点击“运行”即授权执行当前固定代码与 stdin，不重复弹窗。Tutor 也可以在独立的 Turn 级代码授权下自行运行与问题直接相关的最小代码；两条路径互不替代。

## 2. Practice 编程与科学题

- 练习生成表单提供“自动选择合适题型”和“要求包含编程题/科学计算题”。自动模式允许 provider 根据 Lesson objective/evidence 省略不适合题型；明确要求但材料不适合时显示“当前课节缺少可执行/可计算学习目标”，不强行凑题。
- 编程题直接在 Practice 主区完成，包含题干、输入输出、约束、初始代码、公开示例和代码编辑器；不跳转实验室交卷。
- 编程题复用实验室的 CodeMirror 6 工作台和专注编码模式，避免两套编辑体验。
- 交卷后就近显示 queued/running、编译/运行/超时/测试摘要、确定性分数和教学反馈；隐藏测试正文不展示。
- 数学、物理或化学题需要 Wolfram 评分时，在整份答卷的交卷确认中一次说明必要答案可能发送到外部服务，不逐题弹窗。
- execution/Wolfram 不可用时，生成表单禁用对应能力并说明原因；已有题目保留只读，不伪装成本地判题。

## 3. Tutor 代码与科学工具

Tutor 输入区附近增加两个默认关闭的 Turn 级开关：

```text
[ ] 允许本次运行代码（自托管）
[ ] 允许本次使用 Wolfram（外部服务）
```

- 代码开关说明运行发生在自托管隔离服务；科学开关说明本次问题的必要内容可能发送给外部 Wolfram。
- 开启后显示“本次工具调用合计最多 3 次”；不展示内部 Tool 名称和 endpoint。
- 用户发送后立即清除授权；重试同一 Turn 使用原授权快照，新 Turn 必须重新开启。
- Tutor 自主判断是否调用代码、Wolfram 或完全不调用；不要求用户先去实验室运行。
- 回答显示本次实际使用的能力和安全调用摘要；授权但没有实际调用时显示“本次未调用”。
- 工具失败时 Tutor 可以基于课程资料继续回答，但必须明确未获得运行/计算结果，不能伪造。
- Tutor 标题栏增加“专注辅导”图标。展开后历史与问题输入区接近整页宽度，代码块获得足够阅读宽度；返回或 `Escape` 后保留问题草稿、Session、历史滚动和本 Turn 授权。

## 4. 状态与外发

| 能力 | 触发者 | 数据边界 | Agent 自主性 |
| --- | --- | --- | --- |
| 代码实验室 | 用户点击运行 | 自托管执行服务；代码/stdin | 无，模型不能调用 |
| 编程题判题 | 用户交卷 | 自托管执行服务；当前代码与固定 tests | 产品确定性调度 |
| 科学题验证 | 生成/交卷授权 | Wolfram；最小表达式、单位或答案 | 产品受控编排 |
| 代码结果辅助 | 用户逐次勾选后发送 Tutor Turn | Tutor provider；受限结果摘要 | 只能解释摘要 |
| Tutor 代码 | 用户为当前 Turn 开启后发送 | 自托管执行服务；Tutor 生成的最小代码 | 白名单内自主选择，code 最多 2 次 |
| Tutor 科学 | 用户为当前 Turn 开启后发送 | Wolfram；问题所需最小上下文 | 白名单内自主选择，science 最多 3 次 |

Tutor 单 Turn 的 MCP 调用合计最多 3 次。两类 MCP 的 Tool 结果都不直接修改 Memory、掌握度、复习项或课节完成状态。

## 5. 响应式与可读性

- 代码编辑器和输出区可展开到接近整页，并能返回原 Reader 状态。
- 专注编码采用顶部固定工具栏、编辑器/输出双栏；窄视口改为上下堆叠。输出区切换 tabs 不改变编辑器尺寸。
- 窄视口按编辑器、结果、Tutor 顺序堆叠，不并排压缩代码。
- stdout/stderr 使用等宽字体、保留换行、限制高度并可展开；长行横向滚动。
- 任务状态固定在运行按钮附近，不放到页面底部。
- 科学工具开关与发送按钮相邻，但不挤占历史问答区域。
- Practice 编程编辑器在窄视口按题干、编辑器、结果顺序堆叠；正式交卷按钮与运行状态始终就近可见。
- Tutor 问题输入与回答代码块使用等宽代码样式、保留缩进、支持复制和横向滚动；专注辅导模式至少给输入区 240px 稳定高度。
