## 采纳项

OCR 共提出 107 条意见。Codex 结合 Spec 003、ADR 005、已接受的 Stage 3/4 合同、完整仓库实现和现有回归测试进行了复核，采纳了以下高置信问题：

- Skill 运行时错误地把 YAML frontmatter 与正文一起发送给模型，现已改为只传递解析后的正文；不可变快照 hash 仍基于规范化后的完整文件计算。
- Skill 文件在路径解析完成后、正式读取前消失时，可能泄漏底层文件异常，而不是返回稳定的不可用错误。
- 教学回答的 citation ID 接受仅包含空白字符的值。
- Tutor 的公开输入及输出投影 schema 未完整拒绝额外 Session 字段、空白课节标识、未知回答块类型，以及出现在错误块类型上的 certainty。
- Workspace 进入 `deleting` 后，Tutor 的列表、读取、取消和重试路径仍可能回读私有记录。
- Tutor retry 在取得权威锁之前读取原 Turn，存在与删除操作竞争的窗口。
- 检索逻辑假设 embedding provider 一定会返回非空向量列表。
- Redis readiness 在 `ok: false` 时仍可能显示“可用”。
- Reader 切换课程后，Web 可能提交已经失效的课节 ID；已取消的 Skill 能力请求也可能覆盖较新的前端状态。
- 两条回归断言实际无效：Skill hash 断言恒为真，readiness 泄漏断言使用了不可能匹配的大小写；同时移除了一处无意义的测试三元表达式。

## 拒绝或暂缓项

- 登录认证和多用户授权不属于已接受的单用户 self-host Slice 3 合同。Workspace 隔离仍需严格执行；认证能力需要单独经过产品 Gate。
- “Tutor 删除遗漏 Turn/citation”“入队失败后仍保持 queued”“worker 未检查取消”“AgentRun 失败状态从未收敛”等意见，在结合完整 service、worker 和删除图复核后均不成立。
- 删除问答后仍保持 `last_turn_ordinal` 单调递增是有意设计；重新编号会破坏 retry 和幂等身份。
- 拒绝通过关键词过滤用户或 Memory 文本的通用建议。输入已经作为不可信数据进行结构化序列化和约束；按内容关键词过滤会重新引入指导文档禁止的 smoke 专用捷径。
- 同步上传架构、readiness 并行化、无关的历史模型默认值或索引，以及大范围组件重构，属于既有实现或本 Slice 范围之外，不是关闭教学 Skill 合同的必要条件。
- 单文件批次大小、检索问题长度和幂等相关意见，已经由现有产品合同限制，或由经过验证的下层服务统一处理。
