## 采纳 Findings

### High

1. `stage4_eval/runner.py` 的 generation probes 仍 patch
   `call_provider`，与生产 `call_practice_provider` 分流不一致，eval 可能
   误触真实 provider 或失去测试意义。
2. `PracticePanel.submitPaper()` 在 starter code 未编辑时把 coding 题视为
   已回答，随后解引用不存在的 `draft.source_code`，会中断交卷。
3. generation/grading 最终 authority helper 显式禁用 wall check，最后一次
   中间检查后越界的结果仍可能提交，违反既有 10 分钟/3 分钟硬预算。

### Medium

1. Tutor capability 只读取一次，恢复后 UI 永久陈旧；同一处 fetch 没有绑定
   已创建的 AbortController。
2. Tutor 的 per-turn 工具授权与 idempotency key 未在 scope/lesson 变化时
   清理，存在跨上下文继承风险。
3. scratch run 的成功响应有 token 隔离，但迟到 error/finally 仍可覆盖新状态。
4. capability probe 单轮耗时超过 interval 后零等待重试；隔离测试失败时也
   缺少 `finally` 线程清理。
5. eval 的泄漏检查使用恒假表达式。复核 API 合同后确认 `display_text` 是
   用户可见的 Memory 正式字段，不能加入禁止列表；本轮仅移除恒假表达式并
   继续禁止内部 `prompt`。OCR 对 `display_text` 的结论属于误报。

## 暂缓或拒绝

- `TutorPanel.errorMessage` 已正确处理 `Error` 与其他 unknown；相关 finding
  不成立。
- Learning Memory 的 `display_text` 是公开、可编辑 API 字段，不是内部 prompt
  泄漏；禁止它会使正确的 offline eval 失败。
- Practice grading interval effect 依赖 `attempts` 并会清理旧 interval；
  “必然 stale closure”证据不足。
- PracticeJob 幂等已有 workspace 复合唯一约束和 IntegrityError 回读；
  “缺少行锁必然重复创建”不成立。
- enqueue 前 commit 由 reconciler 和 queue-failed 路径覆盖，不视为本轮
  zombie Job 缺陷。
- capability probe engine 是进程级 engine，不是每轮创建；连接泄漏结论
  不成立。
- migration 全表 backfill 是新增 NOT NULL 列的既定迁移步骤，不因 OCR
  建议改为跳过。
- SecretStr 全面改造、Qdrant healthcheck/restart policy、历史表约束、
  DocumentChunk ID、CSS 清理和性能泛化建议超出 Slice 5，留待 Stage 总复盘
  后按独立事实盘点处理。
- source-inspection 测试、重复测试和 dead helper 的清理方向成立，但不与本轮
  High 修复混合扩大范围；Stage 4 收尾需继续补真实 system/eval 覆盖。

## 已实施修复

- generation eval 改为 patch `call_practice_provider`，grading 继续使用
  `call_provider`；同时补齐 grading prompt fixture 的
  `deterministic_verification`。
- generation/grading final authority 现在接收本次执行的 monotonic start 和
  正式 wall limit；generation gate 移到完整 artifact validation 之后、
  `_commit_set` 之前。
- Practice 交卷对所有 draft 使用安全可选访问，未编辑 starter code 可正常提交。
- scratch run 在 Item/source/stdin 变化时失效 token 并清 busy；迟到 success、
  error 和 finally 均不能覆盖新状态。
- Tutor capability 首次、30 秒周期和页面恢复可见时刷新，所有后台请求可取消；
  带工具授权提交前重新确认状态；scope/lesson/workspace 变化清除本 Turn 授权
  与幂等 key。
- capability probe 改为每轮完成后等待 interval，避免慢失败后的零延迟热循环；
  线程测试使用 `finally` 清理。
