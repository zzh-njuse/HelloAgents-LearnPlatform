# Stage 4 Slice 1 OCR Review

Date: 2026-07-16

## Scope and method

OpenCodeReview scanned sanitized full-file copies in five risk blocks with
`deepseek-v4-pro`, `--audience human`, `--concurrency 1`, and `--timeout 10`:

1. practice schema, ORM models, and migration 0016;
2. practice router and orchestration service;
3. worker, generation/grading, queue, AgentRun, and deletion paths;
4. Reader, Tutor, Practice, Practice History, Web API, and CSS;
5. practice tests and Stage 4 eval.

API keys, `.env`, private connection addresses, internal domains, uploaded
source material, sensitive prompts, logs, absolute paths, and provider
configuration were excluded. Provider transport and test connection strings
were redacted. Codex reviewed the excluded prompt/configuration surfaces
locally and performed the cross-block contract review.

## Accepted findings and fixes

- Reject whitespace-only practice idempotency keys at the router boundary.
- Recover concurrent duplicate attempt submissions through the database unique
  constraint, returning an identical replay and rejecting a changed payload.
- Lock practice jobs during retry transitions; `attempt_count` remains a claim
  counter and is incremented only by a successful worker claim.
- Apply complete owner/lease/error/attempt cleanup when an unexpected worker
  error observes `cancel_requested`.
- Reject blank option and short-answer payloads.
- Clear answer drafts when switching practice sets.
- Do not let an unrelated historical degraded set disable generation while no
  set is selected; the API remains the authority for current-source validity.
- Strengthen Stage 4 eval tests so a zero-case hard gate cannot pass vacuously,
  schema rejection checks the precise Pydantic error, and failed real mode is
  checked for report leakage before preview mode runs.

## Findings not applied

- Missing authentication was excluded as a product-boundary false positive:
  this self-host stage has no accepted identity/authorization contract yet;
  workspace scoping is enforced in the current API.
- Claims that application imports were missing, `DocumentChunk.id` lacked a
  default, or tutor acknowledgement timestamps could not be inserted were
  caused by scanning isolated blocks or pre-existing code outside this Slice.
- The answer specification is intentionally stored as private server-side
  grading state. Public projections are explicit allowlists and have negative
  leakage tests; moving or encrypting it is not required by Spec 001/ADR 001.
- Cyclic FK deletion, late-result authority, worker claim/lease behavior, and
  source degradation were already covered by Correction 001-003 and real
  Postgres tests. OCR suggestions to replace the accepted deletion graph or
  weaken exact step-count assertions were rejected.
- General performance indexes, N+1 improvements, older course/workspace
  deletion architecture, broad frontend cancellation refactors, and cosmetic
  lint preferences were deferred because they are not correctness regressions
  introduced by Slice 1.
- `source_degraded` stays fully read-only for the selected historical set, as
  explicitly accepted in the Slice 1 contract.
