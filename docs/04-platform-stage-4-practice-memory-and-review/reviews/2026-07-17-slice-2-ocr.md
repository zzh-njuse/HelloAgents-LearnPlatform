# Stage 4 Slice 2 OCR Review

Date: 2026-07-17

## Scope and method

OpenCodeReview scanned sanitized full-file copies in four risk blocks with
DeepSeek, `--audience human`, `--concurrency 1`, and `--timeout 10`:

1. domain models, migration, schemas, learning service, and projection logic;
2. API lifecycle, jobs, workers, deletion paths, and Tutor integration;
3. learning tests and Stage 4 eval;
4. Review/Memory, Reader, Tutor, Web API, and styles.

The user explicitly approved sending the four sanitized ordinary-source batches.
API keys, `.env`, private connection addresses, internal domains, uploaded source
material, sensitive prompts, logs, absolute paths, and provider configuration
were excluded. Codex reviewed excluded private surfaces locally and performed the
cross-block contract review.

All four scans completed without timeout or skipped files. They covered 27 files
and produced 147 comments:

- domain: 6 files, 41 comments, approximately 583,359 tokens, 16m34s;
- lifecycle: 9 files, 52 comments, approximately 524,520 tokens, 24m49s;
- tests/eval: 6 files, 28 comments, approximately 314,943 tokens, 16m11s;
- Web: 6 files, 26 comments, approximately 284,447 tokens, 18m11s.

Raw output is retained under `reviews/raw/`.

## Accepted findings and fixes

- Preserve the Memory suppression watermark through a full workspace recompute,
  including targets that currently have no negative signals. A dismissed
  tombstone now survives replay, and only enough new independent negative
  evidence can recreate the Memory.
- Detach preserved Memory rows before derived Weakness rows are rebuilt, then
  restore their target-matched Weakness links. This avoids FK failure and keeps
  user-managed Memory intact during recompute.
- Clear Weakness event references before deleting an Attempt's learning events,
  preventing dangling first/last-negative foreign keys.
- Normalize signal and suppression timestamps before comparison so SQLite and
  Postgres timestamp representations behave consistently.
- Ignore stale review-count responses after switching workspaces and cancel
  obsolete requests.
- Stop review-job polling after request errors instead of leaving an interval
  running indefinitely.
- Harden optional review reason and Tutor citation rendering against missing
  fields.
- Serialize Tutor Memory context as explicitly untrusted data and instruct the
  model not to follow instructions embedded in Memory text.
- Add focus-visible states and an appropriate pointer cursor to the new review
  controls.

Regression tests cover suppression through an empty recompute, later reopening
from new evidence, and recomputing while an existing Memory is linked.

## Findings not applied

- Authentication and per-user authorization remain outside the accepted
  self-host identity contract; current APIs enforce workspace/course scoping.
- Claims about nullable practice-set identifiers, missing recompute uniqueness,
  unsupported SQLite partial indexes, and absent chunk IDs conflict with the
  actual ORM and migration constraints.
- Suggestions to discard Tutor idempotency keys after provider failure would
  weaken retry identity and were rejected.
- Course/practice worker findings that ignored the established worker-owned
  retry and lease transitions were cross-block false positives.
- Workspace deletion already rolls back and enters the accepted retry state on
  failure; reversing the external-first deletion order would weaken the
  retriable deletion contract.
- `.hidden { display: none }` intentionally keeps Reader, Tutor, Practice, and
  History components mounted so drafts, selections, feedback, and scroll state
  survive view switches.
- Broad performance refactors, older Slice 1 behavior, and test-style comments
  without a demonstrated Slice 2 regression were not applied.

## Final post-smoke coverage audit (2026-07-18)

The initial OCR pass preceded several material changes discovered during Chrome
smoke, including Lesson Completion, Tutor context synthesis, historical review
navigation, and final scope-count UI fixes. A coverage audit compared the
retained OCR snapshots with the final workspace and found 17 previously scanned
files had changed; migration `0018`, Lesson Completion service code, and other
ordinary source files had not been included in the first pass.

The final ordinary-source state was copied before scanning and divided into 16
risk groups. All groups used DeepSeek, `--audience human`, `--concurrency 1`, and
`--timeout 10`. All 43 copied files were reviewed with non-zero file and token
counts; no group reported skipped files or `token_budget_reached`. Two groups
slightly exceeded ten minutes but completed normally. Actual secrets, `.env`,
uploaded material, logs, internal deployment addresses, provider configuration,
and prompt-definition files remained excluded and were reviewed locally.

High-confidence final fixes:

- add explicit Workspace scope predicates to mastery and weakness lookups and
  course-learning-state deletion;
- recover idempotently from a concurrent Lesson Completion insert using the
  database unique constraint;
- escape citation keys before constructing the feedback display regular
  expression;
- reset a pending Tutor idempotency key when the user changes the question,
  scope, or lesson, while preserving it for an unchanged retry.

Rejected or deferred findings included authentication outside the accepted
self-host contract, already-present database uniqueness, deliberate external-
first retriable deletion, accepted worker ownership patterns, generic N+1
performance refactors, false claims that Tutor calls lacked a timeout, and a
prompt-injection finding that ignored the explicit untrusted-data envelope and
instruction boundary. Broader Tutor teaching quality belongs to Slice 3 rather
than a fixed-question shortcut.

Post-fix verification: 24 focused Learning/Projection/Tutor tests passed; Web
lint and production build passed; Stage 4 offline eval passed 35/35 hard gates
with 3 observational cases. A single-process 164-test run and a later parallel
module attempt both exceeded the local Windows time limit without reporting a
test failure; the pre-smoke full suite baseline remains 162 passed, and all
modules changed by the final OCR fixes passed focused verification. This timeout
is recorded rather than represented as a full-suite pass.
