# Stage 4 Slice 2 前端概念：复习队列与学习 Memory

状态：已于 2026-07-17 通过人工 Gate

## 设计目标

复习是跨课程的重复工作流，不塞进 Practice 单题卡，也不把 Tutor 右栏变成拥挤的万能栏。第一入口是 Workspace 的紧凑“待复习”条；完整操作进入已有 focus page 模式。

## Workspace 入口

```text
┌──────────────────────────────────────────────────────────────┐
│ 今日复习  4 项   2 个已确认薄弱点   1 项等待再次验证  [开始] │
└──────────────────────────────────────────────────────────────┘
```

- 没有 due item 时显示“暂无到期复习”，不占大块空间。
- 点击“开始”打开接近整页的 Review focus page，Escape/返回按钮回到原页面并保留 Reader 状态。

## Review focus page

```text
┌──────────────┬──────────────────────────────────┬────────────────────┐
│ 复习队列     │ 当前复习项                       │ 为什么推荐         │
│              │                                  │                    │
│ [全部 4]     │ 大教堂与市集模式差异             │ 2 次相关作答        │
│ [待复习 2]   │ 状态：需要复习                   │ 单选错误 1          │
│ [待验证 1]   │                                  │ AI rubric 未满足 1  │
│ [稍后 1]     │ 建议：回看课节与原反馈           │ 最后验证：7/17      │
│              │                                  │ [查看依据]          │
│ ● 目标 A     │ [打开课节] [查看反馈] [开始练习] │                    │
│ ○ 目标 B     │                                  │ Memory              │
│ ○ 目标 C     │ [已复习] [稍后⌄] [不适用]       │ 已自动记录          │
│              │                                  │ [编辑] [暂停]       │
└──────────────┴──────────────────────────────────┴────────────────────┘
```

- 左栏是密集队列与筛选，不使用大卡片。
- 中栏只有一个当前任务和明确动作；“已复习”进入等待验证，不显示掌握度上涨动画。
- 右栏解释信号来源，区分“确定性单选”和“AI rubric”；不显示内部 score。
- provisional 使用“初步建议”标识，不能出现“你不掌握”这样的确定结论。

## 学习 Memory 管理

Review focus page 顶部使用两个 tab：`复习队列 | 学习记忆`。

```text
学习记忆                              Tutor 使用 [开关]
────────────────────────────────────────────────────
开源协作模式的边界        active       [编辑] [暂停] [删除]
来源：软件管理 / 2 次有效证据 / 7月17日确认

集市模式与大教堂模式差异  需要复核     [复核] [归档] [删除]
原因：后续练习已显示改善；当前不会发送给 Tutor
```

- “编辑”只编辑用户说明，来源事实单独展示且不可伪造。
- confirmed Weakness 自动出现在“学习记忆”tab，不要求用户逐条点击“记住”；provisional 只显示为初步建议，不创建 Memory。
- 删除使用确认 dialog，说明只删除 Memory，不删除 Practice 历史；后续新的独立证据再次满足阈值时，系统可以重新建立，并明确展示原因。
- 关闭 Tutor 使用时保留列表，明确“不会发送给外部 AI”。

## Reader/Tutor 连接

- Reader 右侧 Review tab 只显示当前 Lesson 的 due/awaiting-validation 摘要，动作跳转 focus page。
- Tutor 回答顶部或输入区附近显示小型状态：“本次将使用 2 条已确认学习记忆”；可点击关闭 Workspace policy。
- 不在 Practice 题目中持续显示 mastery，避免作答时诱导和视觉噪音。

## 状态矩阵

| 状态 | 表现 | 可用动作 |
|---|---|---|
| loading | 固定高度 skeleton | 无 |
| empty | 暂无到期复习 | 查看全部目标 |
| provisional | 初步建议、一次负向依据 | 复习、稍后、不适用；不创建 Memory |
| confirmed | 已确认薄弱点，Memory 已自动记录 | 复习、开始练习、编辑或暂停 Memory |
| awaiting validation | 已复习，等待新 Attempt | 到期后开始练习 |
| resolved | 已有改善证据 | 查看依据、归档 Memory |
| source degraded | 历史只读、来源变化 | 查看历史；不能新验证/注入 Tutor |
| job running | 重算状态条，标明 Workspace | 取消 |
| job failed | 原因与重试 | 重试 |

## 响应式

- `>=1100px`：三栏。
- `720-1099px`：队列窄栏 + 主内容；依据以抽屉打开。
- `<720px`：顶部筛选、单列当前项；依据和 Memory 使用全宽 sheet，不允许横向滚动。
- 长 target、原因和用户说明必须换行，操作按钮保持固定高度，不覆盖内容。
